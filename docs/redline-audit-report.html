Library in Use

@ansonlai/docx-redline-js@^0.1.4
Patches word/document.xml inside the DOCX ZIP with Word revision markup (w:ins / w:del nodes).

Supporting dependencies: jszip (ZIP read/write) · @xmldom/xmldom (DOM impl for Node) · @react-pdf/renderer (PDF fallback)

Download Flow

1
Extract & buffer

POST /api/contracts/extract — parses PDF/DOCX. For DOCX files, the raw buffer is base64-encoded and returned as docxBuffer. Client stores it in sessionStorage('pactora.docxBuffer').

2
Suggest redline

POST /api/contracts/redline — sends clauseText + clauseType. Haiku (or Sonnet + extended thinking for IP/Indemnities) returns alternative clause language split on \nWhy this works:.

3
Accept & store

actions.acceptRedline(clauseType, clauseText, proposedText, explanation) — saves a tuple in the React store. State is synced to localStorage.

4A
Download — DOCX path

If sourceFileType === 'docx' and pactora.docxBuffer is present: POST /api/contracts/redline/export with accepted redlines + base64 buffer. Server decodes → jszip.loadAsync() → extracts word/document.xml → loops through redlines calling applyRedlineToOxml(xml, original, proposed) → reconstructs ZIP → returns blob with correct Content-Type / Content-Disposition.

4B
Download — PDF fallback

If the DOCX export throws or no buffer is available, download-redline-button.tsx silently catches the error and falls back to markup-schedule-pdf.tsx, producing a side-by-side PDF comparison via @react-pdf/renderer.

Current Issues

Critical No DOCX buffer validation before jszip
Buffer.from(docxBuffer, 'base64') is decoded without checking the ZIP magic bytes (50 4B 03 04). Corrupted or truncated sessionStorage data will surface as a cryptic jszip parse error rather than a clear user-facing message.
app/api/contracts/redline/export/route.ts:52
Critical Silent fallback — user unaware DOCX export failed
When the DOCX export throws, the error is caught and the code silently falls back to PDF. No toast, banner, or console warning is surfaced. The user receives a degraded output without knowing it.
components/download-redline-button.tsx:82–85
High sessionStorage quota failure swallowed silently
If the DOCX buffer exceeds the browser's sessionStorage quota (typically 5–10 MB), the write fails silently. Downstream, the download button can't find the buffer and falls back to PDF — again with no user indication that DOCX tracked-changes will be unavailable.
app/api/contracts/extract/route.ts:218–221
High No post-generation XML validation
The output of applyRedlineToOxml() is not checked for well-formed XML before the ZIP is reconstructed. A malformed word/document.xml will corrupt the DOCX silently; Word will either refuse to open it or display a repair prompt.
app/api/contracts/redline/export/route.ts:60–67
High No deduplication for overlapping redlines
The loop processes accepted redlines sequentially with no overlap detection. If two redlines target the same clause text, the second call to applyRedlineToOxml() operates on already-modified XML, which may produce double-wrapped revision markup or mismatched w:ins/w:del nodes.
app/api/contracts/redline/export/route.ts:58–68
Medium Missing Content-Length header
The export response does not set Content-Length. Most browsers infer this from the body, but some proxies and CDN edge nodes may mishandle streaming or show an indeterminate progress indicator.
app/api/contracts/redline/export/route.ts:74–80
Medium Unsafe as unknown as BodyInit type cast
The jszip nodebuffer output is cast via a double assertion rather than typed explicitly as Buffer or Uint8Array. This masks type errors and makes the intent opaque to future maintainers.
app/api/contracts/redline/export/route.ts:74

Next Steps

1
Validate DOCX magic bytes before jszip

Check first 4 bytes of decoded buffer equal 50 4B 03 04; return 400 with a clear message if not. Prevents cryptic crashes from bad sessionStorage data.

2
Surface fallback to the user

Add a toast/banner in download-redline-button.tsx:82–85 when DOCX export fails and PDF is used instead. User should always know what they're downloading.

3
Handle sessionStorage quota gracefully

Wrap the sessionStorage.setItem call in a try/catch; on failure, warn the user that DOCX redline download will be unavailable, or fall back to IndexedDB.

4
Validate reconstructed XML post-generation

Parse word/document.xml with @xmldom/xmldom after applyRedlineToOxml() and check for parse errors before writing back to the ZIP.

5
Add overlap detection to the redline loop

Before the sequential loop, check for duplicate clauseText entries. Warn or merge when two accepted redlines target the same text range to prevent double-revision markup.

6
Add Content-Length to export response

Set Content-Length: ${outputBuffer.length} in the export route headers for proxy and CDN compatibility.

7
Remove double type assertion

Replace as unknown as BodyInit with an explicit Uint8Array cast in export/route.ts:74 to restore type safety.
